# Генератор тестовых вопросов

## Описание

Модуль `question_generator.py` автоматически генерирует тестовые вопросы на основе содержимого текстовых файлов учебника. Вопросы создаются чисто шаблонным методом без использования LLM.

## Как это работает

### Для вкладки "Тест"

Когда приложение запускается с `REBUILD_EMBEDDINGS=true`, сценарий `bootstrapper.py`:

1. Проверяет наличие файла `sources/test_questions.json`
2. Если файл не найден, автоматически генерирует вопросы из текстовых файлов в папке `sources/`
3. Сохраняет сгенерированные вопросы в файл `sources/test_questions_generated.json`
4. Загружает вопросы в базу данных PostgreSQL
5. Генерирует эмбеддинги для правильных ответов

### Типы генерируемых вопросов

Генератор создает вопросы следующих типов:

#### 1. Вопросы определения
- **Формат**: "Что такое [термин]?"
- **Источник**: Определения, извлекаемые из текста
- **Варианты ответов**: 5 синтаксически разных переформулировок определения

#### 2. Вопросы о характеристиках
- **Формат**: "Какие из следующих являются основными характеристиками?"
- **Источник**: Нумерованные списки в тексте
- **Варианты ответов**: Разные способы представления элементов списка

#### 3. Вопросы об отношениях
- **Формат**: "Какое утверждение верно описывает данное отношение?"
- **Источник**: Фразы о связях и отношениях между концепциями
- **Варианты ответов**: Различные описания этого отношения

#### 4. Вопросы на различия
- **Формат**: "Что отличает [опция1] от [опции2]?"
- **Источник**: Альтернативные описания в тексте
- **Варианты ответов**: Различные формулировки различий

## Использование

### Автоматическое генерирование (рекомендуется)

Просто запустите контейнер с флагом `REBUILD_EMBEDDINGS=true`:

```bash
docker compose up --build
```

Вопросы будут автоматически сгенерированы при первом запуске или при пересборке данных.

### Ручное генерирование

Для тестирования или ручного создания вопросов:

```bash
cd backend
python test_generator.py
```

Это создаст файл `../sources/test_questions_generated.json` с автоматически сгенерированными вопросами.

### Генерирование из Python кода

```python
from app.question_generator import QuestionGenerator, generate_questions_from_file

# Способ 1: Прямое использование класса
with open("sources/DM2024_module4.txt", "r", encoding="utf-8") as f:
    text = f.read()

generator = QuestionGenerator(text)
questions = generator.generate_all_questions()

# Способ 2: Функция-помощник
questions = generate_questions_from_file(
    "sources/DM2024_module4.txt",
    "sources/test_questions.json"
)
```

## Структура вопроса

Каждый вопрос имеет следующую структуру JSON:

```json
{
    "id": 1,
    "question": "Что такое высказывание?",
    "answers": [
        "Первый вариант ответа",
        "Второй вариант ответа",
        "Третий вариант ответа",
        "Четвертый вариант ответа",
        "Пятый вариант ответа"
    ]
}
```

## Параметры генератора

Генератор использует следующие параметры (можно настроить в коде):

- `SIMILARITY_THRESHOLD` в `main.py` (0.85) — порог для проверки ответов теста
- Максимум 50 вопросов на одну сессию (ограничение в `bootstrapper.py`)
- Первые 10 определений из текста используются для вопросов-определений

## Примечания

- Генератор работает чисто на основе шаблонов — не требует LLM
- Вопросы извлекаются из текста автоматически по регулярным выражениям
- Каждый вопрос имеет 5 вариантов ответов для совместимости с системой тестирования
- При проверке ответа используется косинусное сходство эмбеддингов (порог 0.85)
- Вопросы сохраняются в БД PostgreSQL с эмбеддингами правильных ответов

## Добавление новых типов вопросов

Для добавления новых типов вопросов отредактируйте класс `QuestionGenerator`:

1. Добавьте метод `generate_xxx_questions()`
2. Реализуйте логику извлечения данных из текста
3. Используйте существующие методы `_generate_*_variants()` для создания вариантов ответов
4. Добавьте вызов вашего метода в `generate_all_questions()`

Пример:

```python
def generate_custom_questions(self) -> List[Dict]:
    """Генерирует вопросы пользовательского типа"""
    questions = []
    # Ваша логика здесь
    return questions
```

## Интеграция с API

API остается неизменным:

- `GET /api/test/generate` — генерирует 5 случайных вопросов из БД
- `POST /api/test/submit` — проверяет ответы на вопросы

Система автоматически использует сгенерированные вопросы, если они загружены в БД.
