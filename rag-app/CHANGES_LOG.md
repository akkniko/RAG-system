# Добавлена функциональность генерации тестовых вопросов

## Дата: 01.06.2026

### Описание изменений

Добавлен модуль для автоматической генерации тестовых вопросов на основе текстовых файлов учебника, используемых для вкладки "Тест".

### Созданные файлы

#### 1. `backend/app/question_generator.py`
- **Размер**: 15.3 KB
- **Назначение**: Основной модуль для генерации вопросов
- **Содержит**:
  - Класс `QuestionGenerator` — основной генератор вопросов
  - Методы для извлечения определений, ключевых терминов и отношений из текста
  - Методы для генерации 4 типов вопросов:
    1. Вопросы определения ("Что такое...")
    2. Вопросы о характеристиках ("Какие из следующих...")
    3. Вопросы об отношениях ("Какое утверждение...")
    4. Вопросы на различия ("Что отличает...")
  - Функция `generate_questions_from_file()` для удобного использования

#### 2. `backend/test_generator.py`
- **Размер**: 2.4 KB
- **Назначение**: Скрипт для тестирования генератора
- **Использование**: `python test_generator.py`
- **Результат**: Создает файл `sources/test_questions_generated.json`

#### 3. `QUESTION_GENERATOR_README.md`
- **Размер**: 7.9 KB
- **Назначение**: Подробная документация генератора
- **Содержит**:
  - Описание как работает система
  - Примеры использования
  - Структура JSON вопросов
  - Инструкции по интеграции
  - Способы добавления новых типов вопросов

### Модифицированные файлы

#### 1. `backend/app/bootstrapper.py`
**Изменения**:
- Добавлен импорт: `from app.question_generator import QuestionGenerator`
- Добавлены новые функции:
  - `generate_test_questions_from_txt()` — генерирует вопросы из текстовых файлов
  - `save_test_questions_from_txt()` — сохраняет сгенерированные вопросы в JSON
- Модифицирована логика загрузки тестовых вопросов:
  - Если `test_questions.json` не существует, автоматически генерирует вопросы
  - Сохраняет сгенерированные вопросы в `test_questions_generated.json`
  - Загружает вопросы в БД и создает эмбеддинги

#### 2. `nginx/entrypoint.sh`
**Изменения**:
- Исправлены переводы строк (с CRLF на LF) для корректного выполнения в Linux контейнере
- Добавлены кавычки вокруг переменных для надежности скрипта

### Исправленные проблемы

1. **nginx контейнер падал при запуске**
   - Причина: `entrypoint.sh` имел неправильные окончания строк (CRLF вместо LF)
   - Решение: Переписан скрипт с правильными UNIX окончаниями

2. **PostgreSQL healthcheck проверял неправильную базу**
   - Причина: Проверял `raguser` вместо `ragdb`
   - Решение: Изменена команда healthcheck на `pg_isready -U ${POSTGRES_USER} -d ${POSTGRES_DB}`

### Как это работает

1. **При запуске контейнеров**:
   ```bash
   docker compose up --build
   ```

2. **bootstrapper.py**:
   - Проверяет наличие `sources/test_questions.json`
   - Если файла нет, генерирует вопросы из файлов в `sources/`
   - Сохраняет результат в `sources/test_questions_generated.json`
   - Загружает вопросы в базу данных
   - Генерирует эмбеддинги для ответов

3. **API endpoints** (без изменений):
   - `GET /api/test/generate` — возвращает 5 случайных вопросов
   - `POST /api/test/submit` — проверяет ответы по эмбеддингам

### Генерируемые вопросы

На основе `sources/DM2024_module4.txt` генерируется примерно:
- **16-50 вопросов** в зависимости от размера текста
- **5 вариантов ответов** для каждого вопроса
- **Различные типы**: определения, характеристики, отношения, различия

### Примеры сгенерированных вопросов

```json
{
    "id": 1,

        "Это утверждение принимающее одно из двух истинностных значений",
        "...",
        "...",
        "..."
    ]
}
```

### Преимущества подхода

✅ **Без LLM** — работает чисто на регулярных выражениях и шаблонах
✅ **Быстро** — генерирует вопросы за секунды
✅ **Надежно** — не зависит от интернета или API
✅ **Настраиваемо** — легко добавить новые типы вопросов
✅ **Автоматично** — интегрировано в процесс инициализации
